Intrroduction :

Le test est une activité importante dont le but est d’arriver à un produit Zéro défaut.
Il est l’exécution ou l’évolution d’un système ou d’un composant par des moyens
automatiques ou manuels, pour vérifier qu’il répond à ses spécifications ou identifier
les différences entre les résulats attendus et les résultats obtenus.
C’est la limite idéaliste vers laquelle on tend pour la qualité du logiciel.
Généralement 40% du budget global est consacré à l’effort de test.
Le test est une recherche d’anomalie dans le comportement de logiciel.
C’est une activité paradoxale :il vaut mieux que ce soit pas la même personne qui développe et qui teste le soft.
D’où le fait qu’un bon test est celuiqui met à jour une erreur (non encore rencontrée).
